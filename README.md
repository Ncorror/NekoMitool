# NekoFlash — Portable OTG Android Flasher

NekoFlash — это Android-приложение для прошивки одного Android-устройства с другого
**без компьютера**. Host-смартфон выступает в роли ADB/Fastboot-хоста через USB OTG,
target-смартфон прошивается напрямую. Реализовано на Kotlin поверх Android USB Host API.

> **Важно:** все действия выполняются на риск пользователя. Прошивка разделов
> `boot`, `init_boot`, `vendor_boot`, `recovery`, `dtbo` может привести к bootloop
> или hard brick, если выбран файл от другой модели, слота или версии прошивки.
> Разблокировка загрузчика Xiaomi выполняется через официальный Mi Unlock API —
> приложение не обходит систему одобрения/квот Xiaomi, разблокировать можно только
> устройство с уже одобренным для этого Mi-аккаунтом.

## Возможности

- **Unlock (Xiaomi Mi Unlock)**: официальная разблокировка загрузчика Xiaomi —
  вход в Mi-аккаунт (WebView), обмен на сервисный токен, подпись запросов к
  Mi Unlock API (nonce/checkClear/requestUnlock) и `fastboot oem unlock` тем же
  протоколом, что использует официальный Mi Unlock Tool. Требует одобренный
  Xiaomi аккаунт — приложение не ускоряет и не обходит одобрение/квоты.
- **Fastboot**: прошивка разделов (boot, init_boot, recovery, vendor_boot, dtbo),
  очередь прошивки, режим перезагрузки, диагностика устройства (getvar),
  определение A/B слотов с прошивкой в оба слота.
- **Xiaomi Fastboot ROM**: анализ и прошивка официальных Fastboot-ROM с разбором
  flash_all-скриптов, блокировкой команд блокировки загрузчика, проверкой product,
  anti-rollback и лимита загрузки.
- **ADB Sideload**: прошивка `.zip` через Recovery → Apply update → Apply from ADB,
  с проверкой SHA-256/MD5.
- **Полный ADB/Fastboot терминал**: интерактивный shell, RSA-авторизация.
- **Профили безопасности**: Новичок / Стандарт / Эксперт с разным уровнем доступа
  к high-risk операциям.
- **Анализ образов**: проверка сигнатур (ANDROID!, VNDRBOOT, AVB0, sparse),
  хэшей и подсказки раздела до прошивки.

## Требования

1. **Host**: Android-смартфон с поддержкой USB OTG.
2. **Target**: Android-смартфон в режиме Fastboot или Recovery / ADB Sideload.
3. Кабель Type-C → Type-C или OTG-переходник (обязательно **data**-кабель,
   не только зарядный).
4. Для прошивки через Fastboot загрузчик target-устройства должен быть заранее
   разблокирован официальным способом (для Xiaomi это можно сделать во вкладке
   **Unlock** самого NekoFlash — см. раздел «Возможности» выше).
5. Файлы прошивки (`.img`, `.zip`, `.tgz`, `.tar`, `.bin` и др.) кладутся в
   `/sdcard/Download/NekoFlash` или импортируются кнопкой **Импорт файла**.
6. Контрольные суммы (`.sha256` / `.md5`) кладутся рядом с файлом.

## Как пользоваться

1. Установите APK на host-смартфон, при первом запуске выдайте доступ к файлам.
2. Импортируйте нужные `.img` / `.zip` кнопкой **Импорт файла** (или положите
   вручную в `/sdcard/Download/NekoFlash`).
3. Переведите target-смартфон в Fastboot или Recovery / ADB Sideload.
4. Подключите target к host по USB OTG, выдайте USB-доступ.
5. На вкладке **Прошить** выберите раздел и файл, либо используйте **Sideload**
   для `.zip`, либо **Мастер Xiaomi-прошивки** для официальных ROM.
6. Логи сохраняются в `/sdcard/Download/NekoFlash/logs/`, отчёты — в
   `/sdcard/Download/NekoFlash/reports/`.

## Индикаторы состояния

- **Режим подключения** в верхней панели: NONE / ADB / FASTBOOT / **FASTBOOTD**
  (userspace, отдельный цвет) — видно, в каком режиме fastboot находится цель.
- **OTG-индикатор** под верхней панелью: активен / нет устройств (проверьте
  кабель или включите OTG) / не поддерживается. Помогает понять, почему цель
  «пропала» — на части телефонов OTG отключается автоматически.
- **Полноэкранный диалог прогресса** при прошивке: процент, скорость, ETA,
  предупреждение не отключать кабель.
- **Подготовка fastbootd**: перед прошивкой Xiaomi ROM, требующей fastbootd,
  приложение предлагает заранее перейти в fastbootd и переподключить OTG —
  чтобы избежать рискованной перезагрузки в середине процесса.

## Безопасность

- Приложение блокирует `fastboot flash`, если загрузчик не разблокирован.
- High-risk операции (vbmeta off, wipe, блокировка загрузчика) требуют профиля
  **Эксперт** и typed-подтверждения.
- Команды блокировки загрузчика в Xiaomi-скриптах блокируются.
- Перед прошивкой выполняется предпроверка (preflight): product, слот, размер,
  сигнатура файла.
