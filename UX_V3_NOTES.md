# UX v3: full-window recovery-style redesign

Цель: уйти от интерфейса «терминал + маленькие вкладки» к приложению, где основные действия выполняются через отдельные полноценные окна и пошаговые сценарии.

## Что изменено

1. Главная страница стала панелью диагностики устройства:
   - текущий режим подключения;
   - product/model;
   - active slot;
   - bootloader unlocked;
   - max-download-size;
   - рабочая папка.

2. Нижняя навигация сокращена до 5 основных окон:
   - Главная;
   - Прошивка;
   - Sideload;
   - Файлы;
   - Консоль.

3. «Сервис» больше не занимает место в нижней панели. Он открывается с главной страницы.

4. Прошивка переработана в очередь:
   - можно добавить boot/init_boot/vendor_boot/vbmeta/recovery/dtbo;
   - пользователь видит список того, что будет прошито;
   - запуск идет одной подтвержденной операцией;
   - есть предупреждение, если имя файла похоже на другой раздел.

5. Добавлен экран активной операции:
   - во время прошивки/sideload приложение показывает отдельное окно;
   - консоль открывается отдельной кнопкой;
   - Stop оставлен только как аварийное действие.

6. Добавлен обычный/экспертный режим:
   - ручной ввод команд скрыт по умолчанию;
   - в экспертном режиме поле команд снова показывается;
   - обычные сценарии прошивки, sideload, файлов и логов работают без экспертного режима.

7. Добавлены дополнительные защиты:
   - очередь и одиночная прошивка блокируются, если Fastboot сообщает bootloader unlocked = no;
   - перед запуском очереди показывается полный список разделов и файлов;
   - vbmeta отдельно помечается как опасный элемент очереди.

## Проверка

- XML-файлы успешно разобраны стандартным XML-парсером.
- Проверено, что все R.id из MainActivity существуют в activity_main.xml.
- Проверено, что все новые строковые ресурсы существуют в values и values-ru.
- Полная Gradle-сборка в sandbox невозможна: wrapper пытается скачать Gradle с services.gradle.org, но интернет недоступен.


## Forum check fixes

- Найдено и исправлено: Fastboot `getvar` часто отдаёт значения через `INFO`, а старая реализация их пропускала.
- Найдено и исправлено: UI очереди показывал `vbmeta`, но протокольный whitelist его не принимал.
- Усилено: блокировка locked bootloader перенесена внутрь `FastbootProtocol.flashPartition`, чтобы прошивка не стартовала даже при обходе UI.
