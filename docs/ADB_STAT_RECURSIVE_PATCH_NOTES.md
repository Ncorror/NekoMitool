# ADB STAT / Recursive Push-Pull Patch Notes — v2.1.8

## Что добавлено

1. `adb pull` теперь сначала выполняет sync `STAT` для remote path.
2. Если remote path — файл, прогресс `pull` считается по точному размеру из `STAT`.
3. Если remote path — каталог, запускается рекурсивный режим:
   - remote-список строится через `cd <dir> && find . -type d -print && find . -type f -print`;
   - локальная структура каталогов создаётся перед скачиванием;
   - каждый файл скачивается через sync `RECV`;
   - `.part` файлы используются до успешного завершения каждого файла.
4. `adb push` теперь принимает не только файлы, но и локальные каталоги.
5. Для рекурсивного `push` remote-каталоги создаются через `mkdir -p`, затем файлы отправляются через sync `SEND`.
6. Для одиночного файла сохранена логика `adb push local.bin /sdcard/Download/` → `/sdcard/Download/local.bin`.

## Ограничения

- Рекурсивный `pull` зависит от наличия `find` в shell target-устройства. На обычных Android-сборках toybox обычно предоставляет `find`, но recovery-среды могут быть урезаны.
- Имена файлов с переводом строки внутри имени не поддерживаются в рекурсивном `pull`, потому что список строится через построчный `find -print`.
- Права владельца/SELinux-контекст не восстанавливаются; передаются данные файлов и структура каталогов.
