# Проверка проекта под форумный Android-host прошивальщик

Цель проверки: приложение не для Google Play Market, а для распространения APK на форуме. Основной сценарий — один Android-смартфон работает как USB-host через OTG и прошивает/диагностирует второй Android-смартфон в Fastboot или Recovery / ADB Sideload без ПК.

## Вывод

Архитектура проекта соответствует цели: приложение использует Android USB Host API, не требует ПК, не запрашивает интернет/контакты/геолокацию/камеру/микрофон, работает с локальной папкой `/sdcard/AdbFastboot`, сохраняет логи и умеет создавать ZIP-отчёт для форума.

## Что хорошо подходит именно для форума

- `uses-feature android.hardware.usb.host` указан как обязательный.
- Протоколы ADB/Fastboot реализуются напрямую через USB bulk endpoints, без внешнего бинарника `adb`/`fastboot` и без ПК.
- `MANAGE_EXTERNAL_STORAGE` оставлен осознанно: для форумной APK-раздачи это нормально, потому что прошивальщику нужно просто читать большие `.img/.zip` из понятной папки.
- Нет `INTERNET`, нет лишних пользовательских разрешений.
- `android:allowBackup` отключён, чтобы логи/отчёты/пути файлов не попадали в Android backup.
- Есть логи, debug-режим и ZIP-отчёт для форума.
- Ручные опасные Fastboot-команды заблокированы whitelist-логикой.
- Есть SHA-256/MD5 sidecar-проверки.
- Есть WakeLock и foreground service для долгих операций.

## Найдено и исправлено

### 1. Fastboot getvar мог неправильно показывать диагностику

Многие bootloader'ы возвращают `getvar` так:

```text
INFOproduct: alioth
OKAY
```

Старая реализация брала значение только из payload финального `OKAY`, поэтому `product`, `current-slot`, `unlocked`, `max-download-size` могли оставаться пустыми.

Исправлено: добавлен разбор `INFO...`-строк в `FastbootProtocol.readGetVarResponse()`.

### 2. Locked bootloader блокировался не на самом нижнем уровне

UI предупреждал/блокировал некоторые сценарии, но сама функция `FastbootProtocol.flashPartition()` могла продолжить работу, если вызов пришёл не через обычный UI-путь.

Исправлено: внутренняя блокировка добавлена прямо в `FastbootProtocol.flashPartition()`. Если `getvar:unlocked = no`, файл не отправляется на устройство.

### 3. Несоответствие UI и протокольного whitelist по vbmeta

UX v3 показывал `vbmeta` в очереди, но `FastbootProtocol.SAFE_FLASH_PARTITIONS` не разрешал `vbmeta`. В результате пользователь мог добавить `vbmeta`, подтвердить очередь, а прошивка остановилась бы уже во время операции.

Исправлено: `vbmeta` добавлен в whitelist, но оставлен как опасный сценарий с предупреждением. В профиле устройства тоже добавлена пометка, что `vbmeta` нужно включать только при явной инструкции для модели.

### 4. Backup приложения был включён

Для форумного прошивальщика лучше не сохранять данные приложения через Android backup, потому что в логах могут быть пути файлов, product/device, VID/PID и другие диагностические строки.

Исправлено: `android:allowBackup="false"`.

## Что ещё стоит сделать перед публичной темой на форуме

1. Собрать debug/release APK на машине с интернетом или с уже установленным Gradle.
2. Проверить минимум на двух host-смартфонах с OTG.
3. Проверить минимум на одном target в Fastboot и одном target в Recovery / ADB Sideload.
4. Приложить к теме форума понятную инструкцию:
   - host-смартфон должен поддерживать OTG;
   - target должен быть заранее переведён в Fastboot или Recovery/Sideload;
   - bootloader должен быть разблокирован официальным способом;
   - приложение не разблокирует bootloader и не обходит защиту производителя;
   - `vbmeta` прошивать только по инструкции для конкретной модели.
5. В теме форума попросить пользователей прикладывать ZIP-отчёт из `/sdcard/AdbFastboot/reports/` при ошибках.

## Проверки в sandbox

- XML-файлы успешно разобраны.
- Все `R.id`, которые используются в Kotlin-коде, существуют в `activity_main.xml`.
- Все строковые ресурсы, которые используются в Kotlin/XML, существуют в `values/strings.xml` и `values-ru/strings.xml`.
- Полную Gradle-сборку APK выполнить нельзя в текущем sandbox, потому что wrapper пытается скачать Gradle с `services.gradle.org`, а интернет недоступен.
